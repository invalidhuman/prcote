## JS의 주요 배열 메서드

1. map : 변환 => 배열
2. filter : 필터링 => 배열
3. reduce : 축소(응축) => 하나의 값 (숫자, 문자열, 객체, 배열)

그 외

- forEach
- find
- some
- every

| **메서드**    | **목적**                                | **반환값**                              | **기존 배열 변경 여부** |
| ------------- | --------------------------------------- | --------------------------------------- | ----------------------- |
| **`forEach`** | 각 요소에 대해 특정 함수 실행           | `undefined`                             | 아님                    |
| **`map`**     | 요소를 일괄적으로 변경 및 변환          | **새로운 배열** (길이 유지)             | 아님                    |
| **`filter`**  | 조건에 맞는 요소만 추출                 | **새로운 배열** (길이 가변)             | 아님                    |
| **`reduce`**  | 요소를 하나로 응축 (합계, 객체 변환 등) | **단일 값** (숫자, 문자, 객체, 배열 등) | 아님                    |
| **`find`**    | 조건에 맞는 첫 번째 요소 찾기           | **요소 하나** (없으면 `undefined`)      | 아님                    |
| **`some`**    | 조건에 맞는 요소가 하나라도 있는지 확인 | `boolean`                               | 아님                    |
| **`every`**   | 모든 요소가 조건을 만족하는지 확인      | `boolean`                               | 아님                    |

## 코테에서의 활용

1. DFS/BFS 탐색 시: 그래프의 인접 노드들을 순회할 때는 **forEach**가 가장 적합.
   단순히 다음 노드로 이동하는 동작이 중요하기 때문입니다.

2. 데이터 가공 시: 입력받은 문자열 배열을 숫자 배열로 바꿀 때는 input.map(Number) 처럼 **map** 사용

3. 조건 탐색 시: 특정 조건을 만족하는 경로가 하나라도 있는지 확인할 때는 forEach를 끝까지 돌리기보다 **some**을 사용하여 중간에 탈출(Short-circuit)하는 것이 효율적
