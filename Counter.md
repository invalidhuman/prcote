## Python Counter 객체의 뺄셈 연산

### 1. collections.Counter의 뺄셈 원리

Counter 객체끼리는 - 연산자를 사용한 뺄셈이 가능합니다. 이때 단순히 키가 빠지는 것이 아니라 **각 키에 대응하는 빈도수(Value)**를 뺍니다.

동작 원리: A - B 연산 시, 결과값의 빈도가 0 이하가 되는 요소는 결과 객체에서 자동으로 삭제됩니다.

용도: 두 집단 간의 빈도 차이를 구할 때 매우 유용합니다.

### 2. 출력 예시 코드

```python
from collections import Counter

# 데이터 준비
A = Counter(['leo', 'kiki', 'eden', 'leo']) # {'leo': 2, 'kiki': 1, 'eden': 1}
B = Counter(['leo', 'kiki'])               # {'leo': 1, 'kiki': 1}

# 뺄셈 연산 수행
result = A - B

print(result)
# 출력: Counter({'leo': 1, 'eden': 1})
# kiki는 1-1=0이 되어 결과에서 삭제됨
```

### 3. 자료구조별 뺄셈(-) 연산 지원 여부

| **자료구조**   | **지원 여부**  | **동작 특징**                                                 |
| -------------- | -------------- | ------------------------------------------------------------- |
| **List**       | **X (불가능)** | `TypeError` 발생. 리스트 간 산술 연산 불가.                   |
| **Set**        | **O (가능)**   | **차집합** 연산. 빈도수와 상관없이 요소의 존재 여부로만 제거. |
| **Dictionary** | **X (불가능)** | 일반 딕셔너리는 객체 간 산술 연산을 지원하지 않음.            |
| **Counter**    | **O (가능)**   | **빈도수 기반 뺄셈**. 연산 후 결과가 0 이하인 키는 자동 삭제. |

### 4. 주의사항 (Set과의 차이점)

Set의 뺄셈: 중복이 없으므로 "A 집합에는 있고 B 집합에는 없는 것"만 남깁니다.

Counter의 뺄셈: 중복(동명이인 등)을 숫자로 기억하고 있기 때문에, "A에 2명 있고 B에 1명 있으면 1명 남기는 것"이 가능합니다.
